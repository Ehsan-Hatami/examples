The following table describes the members of the discovery API

Name Description Data-Type Query Operations
username The username of the customer string n/a
weeksSinceRegistration The number of weeks since the customer had registered to eToro integer min/max
countryId The country that the customer is registered. For a full list of country IDs refer to the Metadata/Countries API call integer in
affiliateId The affiliate ID which is associated with the customer integer in
isPopularInvestor Is the customer a popular investor in the eToro system boolean equality
isFund Does this account represent a fund boolean equality
dailyGain The user's gain in the last trading day number min/max
thisWeekGain The user's gain from the beginning of the trading week (Sunday) number min/max
riskScore The current risk score of the user. The values may be between 0-10. integer min/max
maxDailyRiskScore The maximum daily risk score for a user during this interval integer min/max
maxMonthlyRiskScore The maximum average monthly risk score for a user during this interval integer min/max
trades The total number of trades during this interval integer min/max
topTradedInstrumentId The top traded instrument id during this interval. This value is calculated based on manual trading only. For the full list of instrumens refer to the Metadata/Instruments API integer min/max
topTradedAssetId The top traded asset class id during this interval. This value is calculated based on manual trading only. For the full list of asset classes refer to the Metadata/Instruments API integer min/max
winRatio The winning ratio during the interval. This is the ratio between profitable trades and the total number of trades. It is important to note that a copy trade is treated as a single trade, and the values include open trades number min/max
dailyDD Maximum daily Draw-down during this interval number min/max
weeklyDD Maximum weekly Draw-down during this interval number min/max
peakToValley Maximum peak to vally draw-down during this interval number min/max
profitableWeeksPct The percentage of trading weeks (Sunday to Sunday) which were profitable during this interval number min/max
profitableMonthsPct The percentage of months which were profitable during this interval number min/max
avgPosSize The average trade size percentage relative to the realized equity at the time of opening the trade. number min/max
highLeveragePct The percetnage of trades which used high leverage. number min/max
mediumLeveragePct The percetnage of trades which used medium leverage. number min/max
lowLeveragePct The percetnage of trades which used low leverage. number min/max
copiedTrades The total number of copied trades during this interval integer min/max
copyTradesPct The percentage of copy-trades of all trades executed during this interval number min/max
copyInvestmentPct The investment percenage of copy-trades during this interval. The value is calculated by the total invested amount in copy trades divided by the total invested amount in the interval number min/max
copiers The number of copiers of this user integer min/max
baseLineCopiers The number of copiers of this user one week ago integer min/max
copiersGain The gain in copiers in the last week number min/max
aumTier The tier of the assets under management of this user. This figure is relevant for customers with copiers. The current scale is as following: 1 - up to $50K, 2- between $50-100K, 3 - between $100-500K, 4 - over $500K integer min/max
VirtualCopiers The number of copiers using virtual money to copy the user integer min/max
firstActivity number of days between the beginning of the interval and the first trading activity of the user integer min/max
lastActivity number of days between the last trading activity and the end of the interval integer min/max
activeWeeksPct The percentage of weeks within the interval which the customer had active trades number min/max

A query can be constructed using any one of the members which allows a query operation. Each part of the query is an independant query parameter.

Min/Max

In order to define a minimum value you'll need to suffix the Min to the member name and specify a value. The same applies for a mximum value with a Max suffix

To find customers with at least 10 copiers you need to add a query parameter in the following form CopiersMin=10.

If we would like to find customers with a gain between 10-30% we will add these query parameters: GainMin=10&GainMax=30

IN

In order to define values for the IN operator, they should be comma delimited

If we want to find customers from Germany (country ID 79) and Austria (country ID 13), we will add a query parameter in the following form CountryId=13,79

Equality

The equality type is used for boolean values. In order to use it you need to set a query parameter with a boolean value

If we want to filter for users who are popular investors we will add a query parameter in the following form IsPopularInvestor=true
